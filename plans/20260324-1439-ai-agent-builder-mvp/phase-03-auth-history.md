## Key Insights

- Better-auth manages its own `User` and `Session` tables — do NOT redefine them in Prisma schema. Better-auth uses `better-sqlite3` directly; Prisma manages only app models (`Project`, `Message`). Separate concerns.
- Better-auth's `toNextJsHandler` wraps the auth logic for App Router. Route is `app/api/auth/[...all]/route.ts` — the `[...all]` catch-all handles all auth endpoints (login, logout, register, session).
- Session check in Server Components: use `auth.api.getSession({ headers: await headers() })`. In route handlers: use `auth.api.getSession({ headers: req.headers })`.
- SQLite works fine for dev/local. **Never deploy SQLite to Vercel** — ephemeral filesystem means data is lost on cold starts. Phase 4 handles the migration to PostgreSQL.
- Prisma singleton pattern (`globalForPrisma`) prevents connection pool exhaustion in Next.js dev mode hot-reloads.
- Store only the final `outputDocs` JSON in `Project.output` (TEXT/JSON column) — do not store individual messages in DB for MVP (reduces complexity, Phase 3 scope).

## Requirements

### Functional
- Users can register with email + password
- Users can log in and are redirected to `/chat`
- Authenticated users have projects auto-saved after generation completes
- `/history` page lists all user's projects (name, date, preview of CLAUDE.md first 100 chars)
- Each project row has "Open" (loads into OutputPanel) and "Delete" actions
- Unauthenticated users can still use the chat (generation works without auth); saving is skipped with a "Sign in to save" toast
- Logout button in nav; session expires after 7 days

### Non-Functional
- Auth forms validate client-side before submission
- Passwords never stored in plain text (Better-auth handles bcrypt)
- DB queries complete within 200ms for history page
- Prisma schema versioned via migrations (committed to git)

### Prisma Schema
```prisma
// prisma/schema.prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "sqlite"        // changed to "postgresql" in Phase 4
  url      = env("DATABASE_URL")
}

model Project {
  id          String   @id @default(cuid())
  userId      String
  name        String                          // derived from answers.q1
  answers     String                          // JSON string of answers object
  output      String                          // JSON string of OutputDocs
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}
```
Note: No FK relation to `User` here because Better-auth manages its own user table separately. `userId` stores the Better-auth user ID string for filtering.

### Auth Config (`lib/auth.ts`)
```typescript
import { betterAuth } from "better-auth";
import Database from "better-sqlite3";

export const auth = betterAuth({
  database: new Database("./database.sqlite"),
  emailAndPassword: { enabled: true },
  session: { expiresIn: 60 * 60 * 24 * 7 }, // 7 days
});
```

### Save Project Flow
```
OutputPanel: "Generate" succeeds
  → chatStore.setOutputDocs(docs)
  → if user is authenticated:
      POST /api/projects { name, answers, output }
        → validate session (auth.api.getSession)
        → prisma.project.create(...)
        → return { id }
      → toast("Project saved")
  → else:
      → toast("Sign in to save your project")
```

### History Page (Server Component)
```typescript
// app/history/page.tsx
const session = await auth.api.getSession({ headers: await headers() });
if (!session) redirect('/login');
const projects = await prisma.project.findMany({
  where: { userId: session.user.id },
  orderBy: { createdAt: 'desc' },
});
return <ProjectList projects={projects} />;
```

## Implementation Steps

1. Install deps: `npm install better-auth better-sqlite3 @prisma/client prisma`.
2. Install dev dep: `npm install -D @types/better-sqlite3`.
3. Create `prisma/schema.prisma` with SQLite datasource and `Project` model.
4. Run `npx prisma migrate dev --name init` to create `database.sqlite` and apply migration.
5. Create `src/lib/db.ts` — Prisma singleton.
6. Create `src/lib/auth.ts` — Better-auth config pointing to `database.sqlite`.
7. Create `src/lib/auth-client.ts` — `createAuthClient()` export for client components.
8. Create `src/app/api/auth/[...all]/route.ts` — `toNextJsHandler(auth)`.
9. Create `src/components/auth/LoginForm.tsx` — shadcn Form + Input + Button; on submit call `authClient.signIn.email(...)`.
10. Create `src/components/auth/RegisterForm.tsx` — same pattern for `authClient.signUp.email(...)`.
11. Create `src/app/login/page.tsx` and `src/app/register/page.tsx` — render respective forms.
12. Create `src/app/api/projects/route.ts` — POST: validate session, validate body, `prisma.project.create`.
13. Create `src/app/api/projects/[id]/route.ts` — DELETE: validate session, verify project.userId === session.user.id, `prisma.project.delete`.
14. Update `OutputPanel.tsx` — after `setOutputDocs`, call `POST /api/projects` if session exists; show appropriate toast.
15. Create `src/app/history/page.tsx` — Server Component, fetch projects, render `<ProjectCard>` list.
16. Create `src/components/history/ProjectCard.tsx` — shows name, date, CLAUDE.md preview, Open button (loads into store via client action), Delete button.
17. Update `src/app/layout.tsx` — add simple nav: "AI Agent Builder" logo, History link, Login/Logout button.
18. Test full flow: register → chat → generate → history page shows saved project → delete works.

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| Better-auth + Prisma schema conflict on `User` table | Medium | High | Do NOT add User to Prisma schema; use Better-auth's DB for users only |
| `database.sqlite` not gitignored — accidentally committed | Medium | Medium | Add `database.sqlite` to `.gitignore` immediately |
| Session not available in Server Components | Low | High | Use `auth.api.getSession({ headers: await headers() })` pattern exactly as documented |
| Ownership check missing on DELETE → any user deletes any project | Low | Critical | Always verify `project.userId === session.user.id` before delete |
| Prisma hot-reload in dev creates multiple connections | Medium | Low | Singleton pattern in `lib/db.ts` prevents this |

---

## Security Considerations

- DELETE route MUST verify project ownership (`project.userId === session.user.id`) before deleting.
- POST `/api/projects` MUST validate session; reject 401 if no session.
- `answers` and `output` stored as JSON strings — no SQL injection risk with Prisma parameterized queries.
- Better-auth handles password hashing (bcrypt); never touch password fields manually.
- `database.sqlite` added to `.gitignore` (contains user credentials in dev).
- CSRF: Better-auth handles this via its session cookie configuration.
